# 경상탁구가족 앱 요구사항 명세서

## 사용자 로그인 및 사용자 계정 생성

### 1. 개요

이 섹션에서는 사용자의 로그인 기능에 대한 요구사항을 정의합니다. 이 기능들은 사용자가 서비스를 이용하기 위해 반드시 필요하며, 사용자의 개인정보 보호 및 서비스 이용에 대한 권한 제어를 제공합니다.

### 2. 일반 요구사항

* 시스템은 사용자의 개인정보를 보호해야 합니다.
* 시스템은 사용자에게 로그인 및 로그아웃 기능을 제공해야 합니다.
* 시스템은 비밀번호 재설정 기능을 제공해야 합니다.
* 시스템은 기기 정보를 저장해서 자동 로그인 기능을 제공해야 합니다.

### 3. 상세 요구사항

#### 3.1. 사용자 계정 생성

##### 3.1.1. 사용자 정보 입력

* 관리자는 사용자의 이름, 전화번호, 비밀번호, 학번, 학과 등의 정보를 입력하여 사용자의 계정을 생성할 수 있어야 합니다.
* 일반적인 사용자는 계정을 생성할 수 없어야 합니다.
* 관리자가 입력한 전화번호는 시스템에 이미 등록되어 있지 않아야 합니다.
* 동일한 전화번호 입력시 오류 메시지를 표시받아야 합니다.
* 관리자가 입력한 학번은 시스템에 이미 등록되어 있지 않아야 합니다.
* 동일한 학번 입력시 오류 메시지를 표시받아야 합니다.
* 관리자가 생성한 계정은 시스템에 등록되어야 합니다.

##### 3.1.2. 비밀번호 생성

* 관리자는 비밀번호를 생성하여 사용자의 계정을 생성할 수 있어야 합니다.
* 비밀번호는 8자 이상이어야 합니다.
* 비밀번호는 영문, 숫자, 특수문자를 포함해야 합니다.
* 비밀번호는 대소문자를 구분해야 합니다.
* 비밀번호는 암호화되어 저장되어야 합니다.

##### 3.1.3. 이용약관 및 개인정보 처리방침 동의

* 관리자는 사용자에게 이용약관 및 개인정보 처리방침을 안내하여 동의를 받아야 합니다. (앱의 기능 X)

#### 3.2. 로그인

##### 3.2.1. 전화번호 입력

* 사용자는 등록한 전화번호를 입력하여 로그인할 수 있어야 합니다.

##### 3.2.2. 비밀번호 입력

* 사용자는 등록한 비밀번호를 입력하여 로그인할 수 있어야 합니다.
* 사용자는 '비밀번호 찾기' 기능을 통해 비밀번호를 재설정할 수 있어야 합니다.
* 사용자는 로그인 실패 시 오류 메시지를 표시받아야 합니다.

##### 3.2.3. 비밀번호 재설정

* 사용자는 비밀번호를 재설정할 수 있어야 합니다.
* 관리자는 사용자의 비밀번호를 재설정할 수 있어야 합니다.
* 비밀번호는 8자 이상이어야 합니다.
* 비밀번호는 영문, 숫자, 특수문자를 포함해야 합니다.
* 비밀번호는 대소문자를 구분해야 합니다.
* 비밀번호는 암호화되어 저장되어야 합니다.

#### 3.3. 로그아웃

* 로그인한 사용자는 언제든지 로그아웃할 수 있어야 합니다.

#### 3.4. 자동 로그인

* 사용자는 로그인 시 '자동 로그인' 옵션을 선택할 수 있어야 합니다.
* 시스템은 사용자가 선택한 경우, 다음 로그인 시 자동으로 로그인할 수 있도록 기기 정보를 저장해야 합니다.
* 사용자는 언제든지 자동 로그인 기능을 해제할 수 있어야 합니다.
* 시스템은 자동 로그인 기능을 해제되면 다음 로그인 시 자동으로 로그인되지 않도록 기기 정보를 삭제해야 합니다.

### 4. 용어사전

- **관리자**: 앱을 관리하는 관리자
- **사용자**: 앱을 사용하는 개인
- **전화번호**: 사용자의 휴대폰 번호
- **비밀번호**: 사용자의 비밀번호
- **이용약관**: 사용자가 앱을 사용하기 위해 동의해야 하는 약관
- **개인정보 처리방침**: 사용자의 개인정보를 처리하는 방법
- **기기 정보**: 사용자가 사용하는 기기의 정보
- **자동 로그인**: 사용자가 로그인 시 자동으로 로그인할 수 있도록 기기 정보를 저장하는 기능

## 탁구 경기 기록 및 조회 기능

### 1. 개요

이 섹션에서는 사용자가 탁구 경기를 기록하고 조회할 수 있는 기능에 대한 요구사항을 정의합니다. 이 기능은 사용자가 탁구 경기를 관리하고 기록할 수 있도록 지원합니다.

### 2. 일반 요구사항

* 사용자는 탁구 경기를 기록하고 조회할 수 있어야 합니다.
* 사용자는 탁구 경기를 수정하고 삭제할 수 있어야 합니다.

### 3. 상세 요구사항

#### 3.1. 경기 기록

* 사용자는 참가자, 승패 여부를 입력할 수 있어야 합니다.
* 사용자가 입력한 경기는 자동으로 경기 날짜와 시간이 기록되어야 합니다.
* 입력된 정보는 데이터베이스에 저장되어야 합니다.

#### 3.2. 경기 기록 조회

* 사용자는 모든 경기 기록을 조회할 수 있어야 합니다.

##### 3.2.1. 경기 기록 검색

* 사용자는 경기 기록을 검색을 통해 필터링할 수 있어야 합니다.
* 사용자는 참가자 이름, 경기 날짜, 경기 장소를 검색할 수 있어야 합니다.

#### 3.3. 경기 수정 및 삭제

* 사용자는 자신의 경기 기록을 수정할 수 있어야 합니다.
* 사용자는 자신의 경기 기록을 삭제할 수 있어야 합니다.
* 사용자는 자신이 참여하지 않은 경기 기록을 수정할 수 없어야 합니다.
* 사용자는 자신이 참여하지 않은 경기 기록을 삭제할 수 없어야 합니다.

### 4. 용어사전

- **사용자**: 앱을 사용하는 개인
- **경기**: 탁구 경기를 의미
- **참가자**: 탁구 경기에 참가한 사람 혹은 상대방

## 랭킹 조회 기능 및 상금 조회 기능

### 1. 개요

이 섹션에서는 사용자가 현재 랭킹을 조회할 수 있는 기능에 대한 요구사항을 정의합니다.

### 2. 일반 요구사항

* 사용자는 현재 랭킹을 조회할 수 있어야 합니다.
* 사용자는 상금을 조회할 수 있어야 합니다.
* 사용자는 랭킹 기준을 선택할 수 있어야 합니다.
* 사용자는 랭킹 기준에 따라 랭킹을 조회할 수 있어야 합니다.
* 사용자는 랭킹 기준에 따라 상금을 조회할 수 있어야 합니다.

### 3. 상세 요구사항

#### 3.1. 랭킹 조회

* 사용자는 현재 랭킹에 따라 자신의 점수를 조회할 수 있어야 합니다.
* 사용자는 다른 사용자의 랭킹을 조회할 수 있어야 합니다.
* 사용자는 랭킹 기준을 선택할 수 있어야 합니다.
* 사용자는 랭킹 기준에 따라 랭킹을 조회할 수 있어야 합니다.
* 사용자는 랭킹 기준으로 점수, 게임 수, 승리 수, 패배 수, 점수 폭, 승률 중 하나를 선택할 수 있어야 합니다.

#### 3.2. 상금 조회

* 사용자는 현재 랭킹에 따라 자신의 상금을 조회할 수 있어야 합니다.
* 사용자는 다른 사용자의 상금을 조회할 수 있어야 합니다.

### 4. 용어사전

- **사용자**: 앱을 사용하는 개인
- **랭킹**: 사용자 점수에 따른 순위
- **상금**: 사용자가 동아리 활동을 장려하기 위해 그리고 동아리 활동 참여 시 받는 금액

## 탁구 칠 사람 모집 기능

### 1. 개요

이 섹션에서는 사용자가 탁구를 함께 칠 사람을 모집할 수 있는 기능에 대한 요구사항을 정의합니다.

### 2. 일반 요구사항

* 사용자는 탁구를 함께 칠 사람을 모집할 수 있어야 합니다.
* 사용자는 탁구 칠 사람 모집 공고가 올라오면 알림을 받을 수 있어야 합니다.
* 사용자는 원치 않을 경우 알림을 받지 않을 수 있어야 합니다.
* 사용자는 모집 공고를 생성, 조회, 수정, 삭제(CRUD)할 수 있어야 합니다.


### 3. 상세 요구사항

#### 3.1. 모집 공고 생성

* 사용자는 모집 공고를 **생성**할 수 있어야 합니다.
* 사용자는 모집 공고에 시간, 장소, 참가자 수를 입력할 수 있어야 합니다.
* 사용자는 참가자 수를 최소 1명에서 제한 없음까지 입력할 수 있어야 합니다.

#### 3.2. 모집 공고 조회

* 사용자는 모집 공고를 **조회**할 수 있어야 합니다.
* 사용자는 조회한 모집 공고에서 약속 시간, 약속 장소, 참가자 수를 확인할 수 있어야 합니다.
* 사용자는 참가자가 누구인지 확인할 수 있어야 합니다.
* 사용자는 조회한 모집 공고에서 모집 공고를 참가 신청할 수 있어야 합니다.

#### 3.3. 모집 공고 수정

* 사용자는 모집 공고를 **수정**할 수 있어야 합니다.
* 사용자는 모집 공고에 시간, 장소, 참가자 수를 수정할 수 있어야 합니다.
* 사용자는 참가자 수를 최소 1명에서 제한 없음까지 수정할 수 있어야 합니다.
* 사용자는 자신이 만들지 않은 모집 공고를 수정할 수 없어야 합니다.

#### 3.4. 모집 공고 삭제

* 사용자는 모집 공고를 **삭제**할 수 있어야 합니다.
* 시스템은 모집 공고가 약속 시간이 지나면 해당 모집 공고를 삭제해야 합니다.
* 사용자는 자신이 만들지 않은 모집 공고를 삭제할 수 없어야 합니다.

#### 3.5. 알림 기능

* 사용자는 앱을 키고있지 않아도 모집 공고가 약속 시간이 되면 알림을 받을 수 있어야 합니다.
* 사용자는 모집 공고가 올라오면 알림을 받을 수 있어야 합니다.
* 모집공고를 올린 사용자는 다른 사용자가 모집공고를 참가 신청하면 알림을 받을 수 있어야 합니다.
* 사용자는 알림을 받지 않을 수 있어야 합니다.

### 4. 용어사전

- **사용자**: 앱을 사용하는 개인
- **모집 공고**: 탁구를 함께 칠 사람을 모집하기 위한 게시물
